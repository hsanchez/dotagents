Review the current branch as a first-draft reviewer.

Prioritize correctness, security, error handling, edge cases, compatibility, and meaningful test gaps. Lead with findings ordered by severity and include file and line references whenever possible. If there are no findings, say that clearly and note any residual risk.

Do not approve, merge, push, or modify files.
