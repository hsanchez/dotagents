# Full skill list → SDLC stages

## Stage: Requirements elicitation

Skill(s): clarify, unpack

Note: Two-tier: clarify asks minimal questions; unpack is the explicit
escalation when the user still won't confirm scope after that — deeper,
iterative, produces a Design Brief

## Stage: Analysis / Investigation

Skill(s): research, council, cross-critique

Note: Also tiered: research = single/parallel subagent digging so context
stays clean; council = multiple model perspectives on one question;
cross-critique = second round where those perspectives critique each
other. They're designed to compose.

## Stage: Planning / Design

Skill(s): saga (planning half), unpack (produces the Design Brief)

Note: No standalone "plan" skill — saga seems to own both planning and
execution of an already-agreed plan

## Stage: Implementation

Skill(s): startup, git-guardrails

Note: Session readiness + destructive-command prevention while coding

## Stage: Code Review / QA

Skill(s): audit, review-pr, review-saga, pr-comments, pr-walkthrough

Note: audit = your own diff; review-pr = someone else's PR; review-saga =
same as review-pr but as an explicit, escalation-gated orchestrated
workflow; pr-comments = closing the loop on reviewer feedback;
pr-walkthrough = visualization/onboarding aid

## Stage: Integration / Delivery

Skill(s): create-pr, saga (execution half)

Note: Turning finished work into a mergeable PR

## Stage: Continuity (agent-specific, no classic SDLC analogue)

Skill(s): handoff, resume-handoff

Note: Cross-session state transfer
